# Claude Prompt for Village Water Supply Web App

## ภาพรวมโครงการ
สร้าง Web App ระบบประปาหมู่บ้าน รองรับการใช้งานทั้งบน Mobile และ PC โดยมีผู้ใช้งาน 2 กลุ่มคือ:
- ผู้ใช้น้ำทั่วไป
- ผู้ดูแลระบบ

ระบบต้องสนับสนุนการสมัครสมาชิก ระบบล็อกอิน ตรวจสอบค่าน้ำ ประวัติการใช้น้ำ แจ้งปัญหา และจัดการข่าวสาร พร้อมให้ผู้ดูแลระบบจัดการสมาชิก บันทึกเลขมิเตอร์ คำนวณค่าน้ำ ออกใบแจ้งหนี้ และบันทึกการชำระเงิน

## วัตถุประสงค์หลัก
- ให้ผู้ใช้น้ำตรวจสอบข้อมูลค่าน้ำและประวัติการใช้งานได้เอง
- ให้ผู้ใช้น้ำแจ้งปัญหาและดูข่าวประชาสัมพันธ์
- ให้ผู้ดูแลระบบจัดการสมาชิกและงานทะเบียนค่าน้ำได้ครบวงจร
- ให้ระบบมี UI ที่ทันสมัย ใช้งานง่าย และรองรับทั้ง Mobile/PC

## สแต็ก (Stack)
ให้แนะนำสแต็กที่เหมาะสมสำหรับระบบนี้ โดยเน้น Web App แบบคลาสสิค:
- Frontend: React หรือ Vue, HTML, CSS/SCSS, JavaScript/TypeScript
- Backend: Node.js + Express หรือ NestJS, หรือถ้าต้องการใช้ Python ก็สามารถใช้ Django/Flask
- Database: PostgreSQL หรือ MySQL สำหรับข้อมูลเชิงสัมพันธ์, หรือ MongoDB ถ้าต้องการเอกสารแบบ NoSQL
- Authentication: JWT หรือ session-based auth พร้อมเก็บรหัสผ่านแบบ hash

## คำสั่ง build / dev
ให้เสนอคำสั่งสำหรับการพัฒนาและ build โปรเจกต์ เช่น:
- `npm install`
- `npm run dev` หรือ `npm run start:dev`
- `npm run build`
- `npm run lint`
- `npm test`

ถ้าระบบแยก frontend/backend ให้แยกคำสั่งของแต่ละส่วน เช่น `cd frontend && npm run dev` และ `cd backend && npm run dev`

## Convention
ให้กำหนด convention สำหรับโค้ดและโครงสร้างโปรเจกต์ดังนี้:
- โครงสร้างโฟลเดอร์ชัดเจน: `src/`, `components/`, `pages/`, `routes/`, `controllers/`, `models/`, `services/`
- ชื่อไฟล์และตัวแปร frontend: `PascalCase` สำหรับ component, `camelCase` สำหรับฟังก์ชันและ state
- API route ควรเป็น RESTful เช่น `/api/auth`, `/api/users`, `/api/meters`, `/api/bills`, `/api/payments`, `/api/news`
- ตั้งชื่อ database table/collections แบบชัดเจน: `users`, `meters`, `bills`, `payments`, `news`, `complaints`
- กำหนด style guide เบื้องต้น เช่นใช้ ESLint/Prettier ถ้าเป็น JavaScript/TypeScript
- เขียน commit message แบบสั้นชัด: `feat:`, `fix:`, `docs:`, `refactor:`

## กฎสำคัญ
1. ห้ามเก็บรหัสผ่านเป็นข้อความธรรมดา ต้อง hash รหัสผ่านก่อนเก็บ
2. ต้องแยกสิทธิ์ผู้ใช้งานให้ชัดเจน ระหว่างผู้ใช้น้ำและผู้ดูแลระบบ
3. UI ต้อง responsive และใช้งานได้ดีทั้งมือถือและเดสก์ท็อป
4. ระบบต้องมีการตรวจสอบข้อมูลก่อนเก็บ เช่น ฟอร์มสมัครสมาชิก ต้องตรวจสอบชื่อ, เบอร์โทร, เลขมิเตอร์
5. ข้อมูลสำคัญต้องเก็บในฐานข้อมูลอย่างปลอดภัย และไม่ควรส่งข้อมูลที่ละเอียดอ่อนทาง client โดยตรง
6. ให้ใช้หลักการ single responsibility เพื่อแยกหน้าที่ของ component/module ไม่ให้รวม logic หลายอย่างในที่เดียว
7. ถ้ามีผลการคำนวณค่าน้ำ ต้องแสดงข้อมูลที่เข้าใจง่ายและตรวจสอบได้

## ข้อกำหนดจาก SRS
- หน้า Login สำหรับผู้ใช้น้ำและผู้ดูแลระบบ
- ฟอร์มสมัครสมาชิกเก็บข้อมูล ชื่อ-นามสกุล, ที่อยู่, เบอร์โทร, เลขมิเตอร์น้ำ, รหัสผ่าน
- หน้าแสดงค่าน้ำประจำเดือนและประวัติการใช้น้ำ
- หน้าแจ้งปัญหา
- หน้าแสดงข่าวประชาสัมพันธ์
- หน้าแก้ไขข้อมูลส่วนตัว
- ผู้ดูแลระบบต้องจัดการสมาชิก, บันทึกเลขมิเตอร์, คำนวณค่าน้ำอัตโนมัติ, ออกใบแจ้งหนี้, บันทึกการชำระเงิน, จัดการข่าวสาร, และดูรายงานสรุป

## คำสั่งสำหรับ Claude
ให้ Claude สรุปและออกแบบตามหัวข้อเหล่านี้:
1. สรุปฟีเจอร์หลักของระบบประปาหมู่บ้านจาก SRS
2. ออกแบบสถาปัตยกรรม Web App โดยระบุ frontend, backend, database
3. อธิบายสแต็กที่แนะนำ พร้อมเหตุผลว่าทำไมเหมาะสมกับโปรเจกต์นี้
4. แนะนำโครงสร้างฐานข้อมูลสำหรับสมาชิก, ประวัติการใช้น้ำ, ใบแจ้งหนี้, การชำระเงิน, ข่าวสาร, และการแจ้งปัญหา
5. เขียน user flow สำหรับผู้ใช้น้ำและผู้ดูแลระบบ
6. สร้าง wireframe หรือ mockup คอนเซปต์ UI สำหรับหน้าหลัก, หน้า login, หน้า dashboard, หน้าแจ้งปัญหา, หน้าแก้ไขข้อมูลส่วนตัว, และหน้าจัดการของผู้ดูแล
7. ให้แจ้ง build/dev commands และ conventions ในโปรเจกต์
