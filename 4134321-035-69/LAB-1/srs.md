# Software Requirements Specification (SRS)

## 1. บทนำ

### 1.1 วัตถุประสงค์
ระบบ Web App สำหรับระบบประปาหมู่บ้าน รองรับการใช้งานทั้งบน Mobile และ PC โดยช่วยให้ผู้ใช้น้ำสามารถตรวจสอบค่าน้ำ ประวัติการใช้งาน แจ้งปัญหา และแก้ไขข้อมูลส่วนตัวได้ ขณะที่ผู้ดูแลระบบสามารถจัดการสมาชิก บันทึกเลขมิเตอร์ คำนวณค่าน้ำ ออกใบแจ้งหนี้ บันทึกการชำระเงิน จัดการข่าวสาร และดูรายงานสรุปได้

### 1.2 ขอบเขตของระบบ
- หน้า Login สำหรับผู้ใช้น้ำและผู้ดูแลระบบ
- ฟอร์มสมัครสมาชิก พร้อมจัดเก็บข้อมูล ชื่อ-นามสกุล, ที่อยู่, เบอร์โทร, เลขมิเตอร์น้ำ และรหัสผ่าน
- หน้าตรวจสอบค่าน้ำประจำเดือน
- หน้าแสดงประวัติการใช้น้ำ
- หน้าแจ้งปัญหา
- หน้าแสดงข่าวประชาสัมพันธ์
- หน้าแก้ไขข้อมูลส่วนตัว
- ฟังก์ชันสำหรับผู้ดูแลระบบ ได้แก่ การจัดการสมาชิก, บันทึกเลขมิเตอร์, คำนวณค่าน้ำอัตโนมัติ, ออกใบแจ้งหนี้, บันทึกการชำระเงิน, จัดการข่าวสาร และดูรายงานสรุป
- UI ที่ทันสมัยและใช้งานง่าย

## 2. ข้อกำหนดของผู้ใช้งาน

### 2.1 ผู้ใช้น้ำ
- สมัครสมาชิกและเข้าสู่ระบบ
- ตรวจสอบค่าน้ำประจำเดือน
- ดูประวัติการใช้น้ำ
- แจ้งปัญหาเกี่ยวกับระบบประปาหรือการใช้งาน
- ดูข่าวประชาสัมพันธ์
- แก้ไขข้อมูลส่วนตัว

### 2.2 ผู้ดูแลระบบ
- เข้าสู่ระบบผู้ดูแล
- จัดการสมาชิก
- บันทึกเลขมิเตอร์น้ำของสมาชิก
- คำนวณค่าน้ำอัตโนมัติจากการใช้น้ำ
- ออกใบแจ้งหนี้ให้สมาชิก
- บันทึกการชำระเงิน
- จัดการข่าวสารและประกาศ
- ดูรายงานสรุปการใช้งานและการชำระเงิน

## 3. ข้อกำหนดเชิงฟังก์ชัน

### 3.1 ระบบผู้ใช้
- ระบบต้องมีหน้าจอสมัครสมาชิกที่เก็บข้อมูล:
  - ชื่อ-นามสกุล
  - ที่อยู่
  - เบอร์โทร
  - เลขมิเตอร์น้ำ
  - รหัสผ่าน
- ผู้ใช้งานต้องสามารถเข้าสู่ระบบด้วยบัญชีผู้ใช้น้ำ
- ผู้ใช้งานต้องสามารถดูค่าน้ำประจำเดือน
- ผู้ใช้งานต้องสามารถดูประวัติการใช้น้ำย้อนหลัง
- ผู้ใช้งานต้องสามารถส่งคำร้องแจ้งปัญหา
- ผู้ใช้งานต้องสามารถดูข่าวประชาสัมพันธ์
- ผู้ใช้งานต้องสามารถแก้ไขข้อมูลส่วนตัว

### 3.2 ระบบผู้ดูแล
- ผู้ดูแลระบบต้องสามารถเข้าสู่ระบบด้วยสิทธิ์ผู้ดูแล
- ผู้ดูแลระบบต้องสามารถเพิ่ม แก้ไข หรือลบข้อมูลสมาชิก
- ผู้ดูแลระบบต้องสามารถบันทึกเลขมิเตอร์น้ำของสมาชิก
- ระบบต้องคำนวณค่าน้ำอัตโนมัติจากเลขมิเตอร์
- ระบบต้องสร้างใบแจ้งหนี้สำหรับสมาชิก
- ระบบต้องบันทึกการชำระเงินของสมาชิก
- ระบบต้องจัดการข่าวสารและประกาศ
- ระบบต้องแสดงรายงานสรุปการใช้งานและรายรับ

## 4. ข้อกำหนดเชิงไม่ใช่ฟังก์ชัน
- รองรับการใช้งานทั้งบน Mobile และ PC
- UI ต้องใช้งานง่ายและทันสมัย
- ระบบต้องมีความปลอดภัยในการจัดเก็บข้อมูลผู้ใช้และรหัสผ่าน
- ระบบต้องให้ประสบการณ์การใช้งานที่ราบรื่น

## 5. ข้อกำหนดด้านข้อมูล
- เก็บข้อมูลสมาชิก ได้แก่ ชื่อ-นามสกุล, ที่อยู่, เบอร์โทร, เลขมิเตอร์น้ำ, รหัสผ่าน
- เก็บประวัติการใช้น้ำของสมาชิก
- เก็บข้อมูลการคำนวณค่าน้ำและใบแจ้งหนี้
- เก็บข้อมูลการชำระเงิน
- เก็บข้อมูลข่าวสารประกาศ
